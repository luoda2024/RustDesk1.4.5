# RustDesk 网络协议学习笔记
# 学习时间：2026年5月1日 22:25

## 学习目标
在构建等待期间，系统学习 RustDesk 的网络通信协议，为可能出现的网络相关构建错误做好准备。

## 1. Rendezvous 服务器机制

### 1.1 核心功能
- **服务发现**：客户端通过 Rendezvous 服务器发现对方
- **NAT 穿透**：协助建立 P2P 连接
- **中继备用**：当 P2P 不可用时提供中继服务

### 1.2 协议流程
```
客户端A → Rendezvous服务器 → 客户端B
1. 注册自身信息（ID、公网IP、端口）
2. 查询目标客户端信息
3. 尝试直接 P2P 连接
4. 如果失败，通过服务器中继
```

### 1.3 代码位置
- `src/rendezvous_mediator.rs` - 主要逻辑
- `libs/hbb_common/src/rendezvous.rs` - 协议实现
- `libs/hbb_common/src/protobuf.rs` - 消息格式

## 2. 中继服务器实现

### 2.1 工作原理
- **数据转发**：在两个客户端之间转发音视频数据
- **流量统计**：记录数据传输量
- **负载均衡**：多服务器负载均衡

## 3. P2P 连接建立

### 3.1 NAT 穿透技术
- **STUN**：获取公网 IP 和端口
- **UDP 打洞**：通过中间服务器建立直接连接
- **TCP 穿透**：更复杂的 NAT 穿透技术

### 3.2 实现代码
- `libs/hbb_common/src/nat.rs` - NAT 穿透实现
- `libs/hbb_common/src/udp.rs` - UDP 通信
- `libs/hbb_common/src/tcp.rs` - TCP 通信

## 4. 音视频流传输

### 4.1 视频编码
- **scrap 库**：屏幕捕获和编码
- **H.264/H.265**：视频编码格式
- **硬件加速**：GPU 编码支持

### 4.2 音频处理
- **Opus 编码**：低延迟音频编码
- **噪声抑制**：环境噪声处理
- **回声消除**：避免回声

## 5. 安全机制

### 5.1 加密通信
- **TLS/DTLS**：传输层加密
- **端到端加密**：客户端之间直接加密
- **密钥交换**：安全的密钥协商

### 5.2 身份验证
- **ID/密码**：传统验证方式
- **公钥基础设施**：更安全的验证方式
- **双重验证**：增强安全性

## 6. 可能出现的构建错误及解决方案

### 6.1 网络相关构建错误
1. **依赖缺失**：缺少网络库依赖
   - 解决方案：检查 Cargo.toml 中的网络相关依赖

2. **协议版本不兼容**：protobuf 版本冲突
   - 解决方案：统一 protobuf 版本

3. **平台特定网络API**：Windows/Linux/macOS 网络API差异
   - 解决方案：条件编译处理平台差异

### 6.2 配置相关错误
1. **服务器地址格式错误**
   - 解决方案：验证 config.rs 中的服务器地址格式

2. **端口冲突**
   - 解决方案：检查默认端口是否被占用

3. **SSL/TLS 证书问题**
   - 解决方案：更新证书或禁用证书验证（仅测试）
