# RustDesk 技术学习报告

**学习时间：** 2026-05-02 00:15:28

## 学习内容摘要

### 1. RustDesk 网络架构
- **Rendezvous 服务器**: 使用 UDP 协议进行设备发现和连接建立
- **TCP Hole Punching**: 通过 NAT 穿透技术建立直接 P2P 连接
- **中继服务器**: 当直接连接失败时，通过中继服务器转发数据
- **协议定义**: 使用 Protocol Buffers 定义网络消息格式

### 2. 安全机制
- **公钥认证**: 使用 RSA 公钥进行设备身份验证
- **会话加密**: 建立连接后使用端到端加密
- **权限控制**: 细粒度的控制权限（键盘、剪贴板、文件传输等）
- **许可证验证**: 支持许可证密钥验证机制

### 3. Flutter 与 Rust 集成
- **FFI (Foreign Function Interface)**: Dart 通过 FFI 调用 Rust 函数
- **平台通道**: Flutter 通过 MethodChannel 与原生平台通信
- **异步通信**: 使用 Stream 和 Future 进行异步消息传递
- **跨平台 UI**: 一套代码支持桌面和移动端

### 4. 视频编码与屏幕捕获
- **scrap 库**: 跨平台的屏幕捕获实现
- **硬件加速**: 支持硬件视频编码/解码（hwcodec 特性）
- **多种编码器**: 支持 H.264, H.265, AV1 等编码格式
- **动态码率调整**: 根据网络状况自动调整视频质量

### 5. 构建系统
- **Cargo + CMake**: Rust 和 C++ 代码的集成构建
- **跨平台构建**: 支持 Windows, Linux, macOS, Android, iOS
- **GitHub Actions**: 完整的 CI/CD 流水线
- **签名和打包**: 支持代码签名和多种安装包格式

## 关键发现

### 🔍 发现1: 网络连接优先级策略
- **直接连接优先**: 首先尝试 TCP Hole Punching 建立直接连接
- **中继备用**: 直接连接失败时自动切换到中继服务器
- **NAT 类型检测**: 自动检测 NAT 类型以选择最佳连接方式
- **对项目的影响**: 理解这一点有助于优化 LUODA 的网络连接性能

### 🔍 发现2: 品牌化定制的关键位置
- **二进制名称**: 在 Cargo.toml 和 build.py 中定义
- **应用名称**: 在 Flutter 配置和原生平台配置中
- **图标系统**: 每个平台有独立的图标文件位置
- **服务器地址**: 在 config.rs 和多个配置文件中
- **对项目的影响**: 我们已经成功修改了所有关键位置

### 🔍 发现3: 安全架构设计
- **分层安全**: 网络层、传输层、应用层都有安全措施
- **可插拔加密**: 支持不同的加密算法和密钥交换协议
- **权限隔离**: 不同功能模块有独立的权限控制
- **对项目的影响**: 确保 LUODA 的安全性，防止未授权访问
